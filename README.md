# GRIM Engine

This is the start of an engine to control a Multi-User Dungeon (MUD) game.

This is an experiment, it comes with zero warranty or guarantees, and I'll update this documentation when I reach some kind of milestone.

For context: I come from a MUD called Waterdeep and am trying to recreate the general vibe, but with a more modern twist in Rust.

## Immediate Goals
- ... Learn Rust!
- Telnet Server
- Basic character creation
- Login/Logout
- Who list
- Conversation channels
- Finger command
- Areas and rooms
- Walking between them

## Future Goals
- Websocket Server
- Combat system
- Items, equipment, and inventory
- Drinking potions
- Eating food and pills
- Experience and levels

## Topics I need to solve
- Handling crashes and restarts without losing connections
- Copyover / Builder port stuff
- What kind of storage/database to use for persistence
